7-Técnicas de Modelagem

8-Modelagem de Sistemas - Conceitos

No projeto baseado no sistema do Tinder foram utilazados varios elementos da modelagem de sistemas em busca de representar graficamente e logicamente seu funcionamento. Utilizamos o diagrama BPMN (Business Process Model and Notation) para representar o fluxo de ações desde o cadastro do usuário até a comunicação pós-match no passo 1. Modelamos a estruturas de dados e suas inter-relações utilizando o diagrama de classes UML como observado nos passos 2 e 7.A modelagem também incluiu a formalização de Requisitos Funcionais (RF), Não Funcionais (RNF) e Regras de Negócio (RN) observados no passo 9. Isso assegura que o sistema funcione corretamente, seja eficiente e esteja em conformidade com as políticas estabelecidas (como o match mútuo para liberar o chat).

9-Requisitos de Sistemas - RF, RNF, RN

Funcionais:

  • RF01: Criar conta
  • RF02: Dar like/dislike
  • RF03: Enviar mensagens após match
Não Funcionais:
  • RNF01: Sistema responsivo
  • RNF02: Segurança de dados
  • RNF03: Tempo de resposta inferior a 1 segundo

10-Regra de Negócio

  • RN01: Somente usuários que deram like mútuo podem conversar.
  • RN02: Likes são limitados para usuários não pagantes.
  • RN03: O perfil deve estar completo para ser exibido a outros usuários.
